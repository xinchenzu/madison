## A. Conceptual foundations

### Visual identity as repeatable system

Visual identity is not a single logo. It is a repeatable system: color, type, spacing, image treatment, layout logic, motion/interaction cues, and use rules. It must translate brand strategy into consistent decisions.

**Common misconception:** Visual identity equals taste. The chapter should show that visual choices operationalize audience, archetype, and proof.

**Worked example:** A Sage identity may use restrained color, high contrast, data-forward layouts, and evidence screenshots; a Magician identity may use transformation imagery and reveal interactions.

**Source(s):** TIKTOC.md; local `docs/figures-and-assets.md`.

### Accessibility as brand credibility

WCAG contrast standards give a concrete verification check. WCAG 2.2 Success Criterion 1.4.3 requires at least 4.5:1 contrast for normal text, with exceptions, and 3:1 for large-scale text in related guidance.

**Common misconception:** Accessibility is a compliance afterthought. In a professional brand system, accessibility is proof of care and usability.

**Worked example:** Test foreground/background color pairs in the identity sheet before using them in portfolio sections.

**Source(s):** W3C WCAG 2.2 https://www.w3.org/TR/WCAG22/; W3C understanding contrast https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum

## D. Current state of the field

**Settled:** Consistent visual systems improve recognition and usability; accessibility standards give concrete minimum checks.

**Contested or emerging:** AI image and design tools accelerate asset generation, but originality, rights, accessibility, and consistency remain human-review issues.

**Key references:**
1. W3C WCAG 2.2 — accessibility standard.
2. Tropicana redesign cases — visual recognition risk.
3. Gap redesign cases — logo recognition/governance risk.
4. Local `docs/figures-and-assets.md` — repo asset rules.

**Recent developments:** Generative image systems make it easy to create identity variants, increasing the need for guidelines and provenance.

## E. Teaching considerations

**Where students get stuck:** They choose colors they like instead of colors that carry a strategy and pass accessibility.

**Analogies and framings that work:** "A visual identity is a grammar, not a poster."

**Exercises that build the target skill:** One-page identity sheet with palette, type, image direction, logo direction, layout rules, accessibility check, and misuse examples.
