# media-target-map.md — the pruned, cited media plan (Xinchen Zu)

Aspiration: *"I am becoming a marketing technologist who bridges brand strategy,
data, and AI to create measurable business impact."*

Synthesized from media-research.md. Every kept channel cites the source that
justifies it (rubric line 6). Facts about me are from my resume only.
Numbered sources [1]–[19] are the same list as media-research.md.

## Kept channels

### 1. Online portfolio (case studies) — PROOF
- **Why for my aspiration:** My resume's strongest claims (campaigns, brand work,
  data analysis) are currently just assertions. Employers are skeptical of skill
  lists and want proof of execution shown as outcomes, not outputs. A portfolio is
  where my brand-strategy + data pillars become verifiable. [1][2][3][4]
- **Success looks like:** 2–3 case studies, each framed as problem -> approach ->
  outcome -> recommendation (e.g., a campaign analysis with the business decision it
  drove). [2][3]

### 2. GitHub (+ Kaggle for notebooks) — PROOF
- **Why for my aspiration:** This is where my data + AI pillars get evidenced. My
  resume claims Python "proficient" with nothing shown; GitHub/Kaggle is the field's
  standard place to host code and notebooks, and it's exactly where my top-gap AI
  project would live as a runnable artifact. [5][6][8]
- **Success looks like:** A well-presented repo using a real, messy dataset with
  documented cleaning and a business recommendation — not tutorial staples (Titanic/
  Iris); presentation good enough that a hiring manager spends 3 minutes, not 30
  seconds. [7]

### 3. MarTech.org community + MarTech Conference (+ MarketingProfs) — DISCOVERY + VOICE
- **Why for my aspiration:** This is the named ecosystem for *marketing
  technologists* specifically — the exact hybrid role I'm becoming — rather than a
  generic marketing crowd. It's where the field's conversation and peers are. [12][13][14][15]
- **Success looks like:** Active participation (reading, then contributing); over
  time, a comment/contribution or conference session that puts my work in front of
  the field. [14]

### 4. LinkedIn — DISCOVERY (included on evidence, not habit)
- **Why for my aspiration:** Not "everyone should use LinkedIn" — included because
  93% of recruiters source candidates there and search by skills/keywords, so for a
  role-seeker it's the highest-probability discovery surface. Its job in my system is
  narrow: be findable for my target keywords and route to my portfolio/GitHub. [18][19]
- **Success looks like:** Profile surfaces in recruiter keyword searches for my
  target terms; inbound recruiter contact; clickthroughs to the portfolio. [18][19]

### 5. Certifications — Google Analytics 4, HubSpot — PROOF (supporting)
- **Why for my aspiration:** They are costly-to-fake signals of tool proficiency for
  the "data/martech" side of my aspiration; GA appears in 41% of marketing-analyst
  postings, and HubSpot demand is high. Kept as *supporting* proof, not a headline. [9][10][11]
- **Success looks like:** GA4 certified; the credential paired with a portfolio
  project that actually uses the tool (a cert alone is "rarely enough"). [9][10]

### 6. Data-driven storytelling content — VOICE
- **Why for my aspiration:** Credibility for this role is built through data-driven
  storytelling, not posting volume. It lets me demonstrate analytical thinking, which
  is the through-line of my aspiration. [1]
- **Success looks like:** Occasional posts/write-ups that each turn one analysis into
  a clear, decision-oriented story — quality and reasoning over frequency. [1]

## Considered but rejected (each keeps its reason)

| Channel considered | Rejected because | Source |
|---|---|---|
| Generic social broadcasting / "be active online" (e.g., Instagram/TikTok personal-brand posting) as a primary strategy | Employers in this field reward proof of execution over presence; volume without evidence doesn't build the credibility the role needs. | [1][2] |
| Certifications as the PRIMARY/sole strategy | A cert (e.g., HubSpot) on its own is "rarely enough" to land a serious role without demonstrable skills + a portfolio. Kept only as supporting proof. | [10][11] |
| Tutorial-staple / popular Kaggle clone projects (Titanic, Iris) on GitHub | Hiring managers discount them; they signal coursework, not capability. Use real, messy datasets instead. | [7] |
| GitHub star-chasing / quantity of repos as the signal | The signal is a well-presented project a manager can evaluate, not repo count or popularity. | [7] |
