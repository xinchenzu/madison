# Cowork Prompt: Extract Supplemental Sections from Chapters
## For "AI + 1" Textbooks — *Irreducibly Human* Series

---

## ROLE & CONTEXT

You are working on an "AI + 1" textbook in the *Irreducibly Human* series. You have access to all chapter markdown files. Your job is to:

1. Scan the book directory for any folder containing 8 or more chapter files
2. For each qualifying book, create five extraction directories alongside `chapters/`
3. Extract any existing supplemental sections from chapter files into consolidated markdown files in those directories
4. Leave a labeled placeholder where a section is absent — do not generate content

This is extraction only. You are not writing or improving any content. Instructor choice governs what exists.

---

## STEP 1 — IDENTIFY QUALIFYING BOOKS

Scan the project directory. A folder qualifies if it contains a `chapters/` subdirectory with **8 or more `.md` files**.

For each qualifying book:
- Read `metadata.yaml` (if present) and extract the `title:` field
- If no `metadata.yaml` or no `title:` field, use the folder name as the title slug
- Normalize the title to a filename slug: lowercase, spaces to hyphens, no special characters

---

## STEP 2 — CREATE EXTRACTION DIRECTORIES

For each qualifying book, create the following directories **at the same level as `chapters/`**:

```
[book-folder]/
  chapters/          ← existing, do not modify
  images/            ← existing, do not modify
  wayback/           ← NEW
  when-to-use-ai/    ← NEW
  llm/               ← NEW
  cli/               ← NEW
  ai-validation/     ← NEW
```

Do not create these directories if they already exist. Do not modify any existing files.

---

## STEP 3 — EXTRACT SECTIONS FROM CHAPTERS

For each chapter file in `chapters/`, scan for the following section headings. Extraction is by heading match only — do not interpret, summarize, or rewrite content.

| Directory | Section heading to match |
|---|---|
| `wayback/` | `## AI Wayback Machine` |
| `when-to-use-ai/` | `### Exercise 1 — When to Use AI` |
| `llm/` | `### Exercise 3 — LLM Exercise` |
| `cli/` | `### Exercise 4 — CLI Exercise` |
| `ai-validation/` | `### Exercise 5 — AI Validation Exercise` |

**Extraction rule:** Extract from the matched heading through to the next heading of equal or higher level (`##` or `###`), or end of file — whichever comes first. Include the heading line itself.

**If a section is absent from a chapter:** insert a placeholder (see format below). Do not skip the chapter entry.

---

## STEP 4 — WRITE CONSOLIDATED FILES

For each extraction directory, write a single consolidated markdown file:

**Filename:** `[title-slug]-[dirname].md`

Examples:
- `wayback/intelligence-wayback.md`
- `when-to-use-ai/intelligence-when-to-use-ai.md`
- `llm/intelligence-llm.md`
- `cli/intelligence-cli.md`
- `ai-validation/intelligence-ai-validation.md`

**File structure:**

```markdown
# [Book Title] — [Section Type]

> Extracted from `chapters/`. Each entry corresponds to one chapter file.
> Sections are instructor-authored. Missing sections show a placeholder only.
> Do not edit this file directly — edit the source chapter file, then re-run extraction.

---

## Chapter [N]: [Chapter Title]
*Source: `chapters/[filename].md`*

[extracted section content here]

---

## Chapter [N]: [Chapter Title]
*Source: `chapters/[filename].md`*

[extracted section content here]

---
```

**Placeholder format** (used when a section is absent from a chapter):

```markdown
## Chapter [N]: [Chapter Title]
*Source: `chapters/[filename].md`*

> **Section not yet authored.** No `[Section Heading]` block found in this chapter file.
> To add this section, edit the source chapter file directly.

---
```

**Chapter ordering:** Match the order of files in `chapters/` — sort by filename alphabetically, which should match chapter numbering if files follow standard naming conventions (`01-`, `02-`, etc.).

---

## STEP 5 — REPORT

After writing all files, produce a brief extraction report:

```

Directories created: [list any newly created]
Directories skipped (already existed): [list any skipped]

| Ch 01: [title] | ✓ | ✓ | ✓ | — | ✓ |

Legend: ✓ = extracted  — = placeholder inserted

Files written:
- wayback/[title-slug]-wayback.md
- when-to-use-ai/[title-slug]-when-to-use-ai.md
- llm/[title-slug]-llm.md
- cli/[title-slug]-cli.md
- ai-validation/[title-slug]-ai-validation.md
```

---

## RULES

- **Read only from `chapters/`.** Do not read from any other directory.
- **Write only to the five extraction directories.** Do not modify any chapter file.
- **Never generate content.** If a section is missing, the placeholder is the correct output.
- **Preserve extracted content exactly.** No reformatting, no summarizing, no cleanup.
- **One run per book.** If the consolidated file already exists in an extraction directory, overwrite it — it may be stale from a previous pass.
- **No emojis** in any output file.
- Chapter attribution (`*Source: ...*`) must appear on every entry, present or placeholder.

---

## HEADING REFERENCE

The five section headings this prompt looks for are:

```
## AI Wayback Machine
### Exercise 1 — When to Use AI
### Exercise 3 — LLM Exercise
### Exercise 4 — CLI Exercise
### Exercise 5 — AI Validation Exercise
```

Note: Exercise 2 (When NOT to Use AI) is always paired with Exercise 1 under the same `when-to-use-ai/` extraction. If `### Exercise 2 — When NOT to Use AI` immediately follows Exercise 1 in the source file, include it in the same extracted block.

---

## NOTES FOR ADAPTING

- **Multiple books in one repo:** This prompt will process every qualifying folder it finds. If you want to limit it to one book, specify the folder name explicitly at the top of your Cowork session.
- **Non-standard chapter filenames:** If your chapters don't use numeric prefixes, add a note to your Cowork session specifying the sort order.
- **Re-running after new chapters are added:** Safe to re-run at any time. Consolidated files are overwritten; no chapter files are touched.
